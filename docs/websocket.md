# WebSocket

Этот документ описывает текущий WebSocket-контракт, реализованный в `config/websocket`.

## Назначение

WebSocket-канал используется только для доставки событий чатов от сервера к клиенту. Бизнес-операции выполняются через HTTP API, а изменения состояния приходят через STOMP.

Текущая конфигурация:

- STOMP endpoint: `/ws`
- подписка для пользователя: `/user/queue/events`
- префикс для брокера сообщений: `/queue`
- префикс для пользователя: `/user`

## Подключение

URL подключения:

```text
ws://<host>:<port>/ws?token=<jwt>
```

### Аутентификация

JWT передается только через query-параметр `token`.

Сервер:
- извлекает `token` из query-параметра
- парсит и валидирует JWT
- использует `sub` как идентификатор пользователя

Подключение отклоняется, если:

- отсутствует `token`
- токен не удается распарсить
- токен истек
- `iss` не совпадает с `server.jwt.issuer`
- `aud` не содержит `server.jwt.audience`
- `sub` отсутствует или пустой

Ожидается, что в `sub` находится UUID пользователя.

## Подписка

После успешного подключения клиент подписывается на:

```text
/user/queue/events
```

Это персональная очередь пользователя. На стороне сервера события отправляются через:

```java
convertAndSendToUser(userId, "/queue/events", event)
```

## Формат события

Каждое событие использует единую обертку:

```json
{
  "type": <message_type>,
  "chat_id": <chat_id>,
  "payload": {}
}
```

Поля:
- `type`: тип события
- `chat_id`: идентификатор чата
- `payload`: полезная нагрузка, зависит от `type`

## Типы событий

| `type` | `payload` | Значение |
| --- | --- | --- |
| `CHAT_CREATED` | `ChatDto` | чат создан |
| `CHAT_UPDATED` | `ChatDto` | чат обновлен |
| `CHAT_DELETED` | `ChatDto` | чат удален |
| `MESSAGE_SENT` | `ChatMessageDto` | сообщение отправлено |
| `MESSAGE_EDITED` | `ChatMessageDto` | сообщение изменено |
| `MESSAGE_DELETED` | `long` | сообщение удалено |
| `MEMBER_JOINED` | `ChatMemberDto` | участник добавлен в чат |
| `MEMBER_LEFT` | `UUID` | участник вышел или был удален |

## Семантика доставки

- `CHAT_CREATED` отправляется владельцу и всем исходным участникам.
- `CHAT_UPDATED` отправляется текущим участникам чата.
- `CHAT_DELETED` отправляется всем пользователям, которые были участниками до удаления.
- `MESSAGE_SENT`, `MESSAGE_EDITED` и `MESSAGE_DELETED` отправляются текущим участникам чата.
- `MEMBER_JOINED` отправляется текущим участникам чата.
- `MEMBER_LEFT` отправляется оставшимся участникам чата.
